# Баг репорт

## Баг 1: Отсутствует валидация отрицательной цены

**ID теста:** TC-007  
**Заголовок:** API принимает объявления с отрицательной ценой  
**Серьезность:** High  
**Приоритет:** High  

**Шаги воспроизведения:**
1. Отправить POST запрос на /api/1/item
2. В теле запроса указать отрицательную цену: `"price": -100`
3. Получить ответ

**Ожидаемый результат:** 
- Статус код: 400 (Bad Request)

**Фактический результат:**
- Статус код: 200 (OK)
- Объявление успешно создано

**Дополнительная информация:**
- Система должна предотвращать создание объявлений с нереалистичными ценами

## Наблюдение: Несогласованность кодов ошибок для невалидных ID

**Контекст:** Невалидные тесты 
**Затронутые тесты:** TC-019

**Наблюдение:**
В тестовых данных ожидаются разные статус коды для сходных ситуаций с невалидными данными:
- В остальных тестах: ожидается статус 400
- TC-019: ожидается статус 404
